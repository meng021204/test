# 雷达信号处理赛题说明

本文档说明如何准备并提交雷达目标检测答案。
## 1. 数据使用

### 1.1 数据参数说明
- 数据维度：300 脉冲 × 8000 快时间采样点
- 数据类型：复数 I/Q 数据
- 数据排列：
行：慢时间（脉冲，第 1 维：300）
列：快时间（距离门，第 2 维：8000）
- 载波频率：fc = 150×10^6 Hz
- 发射信号形式: 线性调频信号
- 信号带宽：B = 5 MHz
- 脉冲宽度：Tp = 20 us
- 脉冲重复频率：PRF = 500 Hz
- 快时间采样率：fs = 10 MHz
- 光速：c = 3×10^8 m/s

### 1.2 目标范围说明
- 距离：65~95 km
- 速度：-800~-1000 m/s
- 加速度：-85~-100 m/s^2


## 2. 提交格式

### 2.1 文件要求
- **格式**：CSV 文件
- **编码**：UTF-8
- **文件名**：建议为 `Target_submission.csv`

### 2.2 表头与列
必须包含以下表头与列（顺序、名称需与示例一致）：

| 列名 | 含义 |
|------|------|
| 目标序列 | 目标编号 |
| 距离 | 目标距离（数值） |
| 速度 | 目标速度（数值，可负） |
| 加速度 | 目标加速度（数值，可负） |

### 2.3 示例（正确格式）
```csv
目标序列,距离,速度,加速度
1,100.5,50.2,-10.0
2,200.0,-60.3,12.5
3,300.8,-55.0,-8.2
4,150.2,-48.6,-11.0
```

### 2.4 数值要求
- 距离、速度、加速度均为**数值**，不允许出现 `xx`、空值或非数字字符

### 2.5 常见错误
❌ 将占位符 `xx` 保留在文件中  
❌ 使用中文标点或空格分隔  
❌ 缺少表头或列名错误  
❌ 编码非 UTF-8 导致乱码

# 雷达数据处理赛题说明

## 1. 任务简介

参赛者需要从给定的“点迹级”原始数据中，**自动提取目标航迹**，并将结果以规定格式的 CSV 文件提交。  
主办方会使用统一的打分脚本 `score_tracks.py` 和标准答案文件，对参赛结果进行自动评测。

## 2. 提供的数据与脚本

- **训练/评测数据**
  - `Track_A_dataset.csv`
  - `Track_B_dataset.csv`

- **标准答案**（由主办方提供，仅用于打分）
  - `Track_A_GT.csv`
  - `Track_B_GT.csv`


参赛者需要做的是：根据 `Track_A_dataset.csv`、`Track_B_dataset.csv`，生成符合格式要求的提交文件 `Track_A_submission.csv`、`Track_B_submission.csv`。

## 3. 提交文件列表与命名约定

参赛时需提交 **两个 CSV 文件**（不压缩或按主办方要求打包）：

- `Track_A_submission.csv`
- `Track_B_submission.csv`

文件应使用 **UTF-8 编码**，**首行为表头**。

## 4. 提交 CSV 格式要求

### 4.1 必选列（必须存在）

每一行对应一个“提取出的航迹点”。两份提交文件均需包含以下 **5 列**（表头名称需完全一致，大小写一致，无多余空格）：

1. **`航迹批号`**
   - 含义：该点所属航迹的编号（整数）。
   - 要求：
     - 同一条航迹的所有点，`航迹批号` 必须相同；
     - 不同航迹建议使用不同的正整数编号（1, 2, 3, ...），具体编号值可以由参赛者自行定义。

2. **`fpga时间`**
   - 含义：该点对应的 FPGA 时间，单位为毫秒（ms）。
   - 要求：
     - 为数值型（可为整数或浮点数）；
     - 同一条航迹内，时间序列应与实际运动过程一致（建议按时间递增）。

3. **`距离`**
   - 含义：目标与雷达的距离，单位：米（m）。
   - 要求：数值型。

4. **`方位`**
   - 含义：目标的方位角，单位：度（deg）。
   - 要求：
     - 数值型；
     - 与标准答案同一坐标系下的角度（与提供数据中“点迹方位”“方位”等字段含义一致）。

5. **`俯仰`**
   - 含义：目标的俯仰角，单位：度（deg）。
   - 要求：数值型。

### 4.2 可选列（可以存在但不会参与计分）

若需要，参赛者可以在 CSV 中额外保留以下列，这些列 **不会参与计分**，仅会被打分脚本忽略：

- `航速`
- `航向`
- `径向速度`

只要 **5 个必选列存在且命名正确**，多出的列不会影响打分结果。

## 5. 提交文件示例

仅展示必选列（实际文件可包含更多列）：

```text
航迹批号,fpga时间,距离,方位,俯仰
1,573585,3793,76.22,24.09
1,573594,4087,77.74,-3.52
2,573624,2594,80.45,-2.10
2,573647,710,83.34,30.54
...
```

## 6. 注意事项

- **编码**：请使用 UTF-8 编码保存 CSV（推荐带 BOM 的 `utf-8-sig`，以兼容常见表格软件）。
- **表头**：必须包含首行表头，且列名与本说明一致。
- **数据质量**：
  - 一条物理航迹建议输出多帧点迹；
  - 同一航迹内的时间必须合理（建议非乱序）；
  - 航迹编号只需在各自文件内自洽，不要求与主办方编号一致，打分时会自动匹配。

# 目标类别识别赛题说明

## 1. 数据说明

- **训练数据目录**：`classification_dataset_split/`
  - 目录结构按类别划分，每个类别一个子目录，例如：

    ```text
    classification_dataset_split/
      Bird/
      Fixed-wing UAV/
      Helicopter/
      Passenger ship/
      Rotary drone/
      Speedboat/
    ```

  - 各类别下的目录包含该类别的训练集。
  - 这些数据仅用于**模型训练与调参**，不会直接参与打分；打分只依赖 `classification_test_data/` 。

- **测试特征数据目录**：`classification_test_data/`
  - 文件形式：`test_data_1.csv`、`test_data_2.csv`、…、`test_data_18.csv`
  - 每个 CSV 对应一段雷达回波样本，包含若干帧观测。
  - 表头及字段：

    ```csv
    A_m,E_m,R_m,SNR,V_m
    ```

    - **A_m**：方位测量值（角度，单位 deg）
    - **E_m**：俯仰测量值（角度，单位 deg）
    - **R_m**：距离测量值（单位 m）
    - **SNR**：信噪比（单位 dB）
    - **V_m**：速度测量值（径向速度，单位 m/s）



## 2. 参赛者提交答案格式

参赛者需要在**项目根目录**提交一个预测结果文件：`predictions.csv`。

- **文件名**：`predictions.csv`
- **编码**：UTF-8
- **列及格式**：

  ```csv
  name,label
  test_data_1,Bird
  test_data_2,Speedboat
  ...
  ```

  - **name**：测试样本名，CSV 文件名（不含扩展名），例如  `test_data_1`
  - **label**：样本所属类别名称，目前类别包括：
      - `Bird`
      - `Fixed-wing UAV`
      - `Helicopter`
      - `Passenger ship`
      - `Rotary drone`
      - `Speedboat`

    模型预测的类别名称，需在上述类别集合之内。

> 注意：
> - 不能遗漏任何一个标准答案中的 `name`，否则会被记为缺失预测；
> - 不要额外增加不存在于标准答案中的样本名，否则会被计为多余预测。


- **评分规则简述**：
  - 若标准答案共有 \(N\) 个样本，其中 \(K\) 个被正确分类，则准确率 \(acc = K / N\)；
  - 最终得分 `score = round(acc * 100)`，并限制在 0–100 区间。
